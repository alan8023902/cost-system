---

1. Load `docs/Frontend-UI-Global-Standard.md` before changing UI code.
2. Enforce workbench layout: Header + LeftNav + MainContent.
3. Keep dense, professional, data-oriented UI. Avoid SaaS marketing styles.
4. Prefer table-driven pages for list/detail management.
5. Keep indicators read-only and make version/permission lock states explicit.
6. Place lifecycle actions in page headers only.
7. Use 4px corners, subtle borders, minimal shadow, left-aligned layout.
8. Keep spacing on a strict 8px scale (4/8/16/24/32).
